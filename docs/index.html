Producto seleccionado

Focalboard

Focalboard es una plataforma open source de gestión de proyectos y tareas, basada en tableros, tarjetas y flujos de trabajo colaborativos.

El sistema permite organizar actividades, administrar tableros de trabajo, gestionar tareas y facilitar la coordinación entre integrantes de un equipo.

Problema que resuelve

Permite centralizar tareas, reducir la dispersión de información y mejorar el seguimiento del avance de proyectos.

Usuarios objetivo

Equipos de desarrollo, grupos académicos, gestores de proyectos y usuarios que necesitan organizar actividades colaborativas.

Valor del producto

Mejora la visibilidad del trabajo, facilita la colaboración y permite organizar tareas mediante una interfaz visual.

Funcionalidades principales

Gestión de tableros

Creación y administración de espacios de trabajo colaborativos.

Gestión de tarjetas

Registro, edición y seguimiento de tareas dentro de los tableros.

Vistas y filtros

Organización de información mediante vistas, agrupamientos y filtros.

Miembros y permisos

Administración de usuarios, roles y responsabilidades dentro del sistema.

Plan de pruebas unitarias

Este plan corresponde al Hito 1 y se enfoca únicamente en la planificación. No se presentan casos de prueba ejecutados ni resultados de pruebas.

Objetivo general

Definir un plan de pruebas unitarias para Focalboard que permita organizar la validación de componentes individuales del sistema, estableciendo una base metodológica para actividades posteriores de control de calidad.

Objetivos específicos

  • Identificar los módulos principales del sistema.
  • Delimitar el alcance inicial de las pruebas unitarias.
  • Definir herramientas y tecnologías previstas.
  • Establecer una estrategia general para backend y frontend.
  • Documentar la planificación inicial mediante GitHub Wiki.

Alcance de las pruebas

Componentes incluidos

  • API REST.
  • Lógica principal de aplicación.
  • Gestión de tableros.
  • Gestión de tarjetas.
  • Gestión de miembros y permisos.
  • Componentes React principales.

Componentes excluidos

  • Pruebas de integración.
  • Pruebas end-to-end.
  • Pruebas de rendimiento.
  • Pruebas de carga.
  • Pruebas de seguridad avanzada.
  • Infraestructura de despliegue.

Estrategia de pruebas unitarias

Enfoque

Las pruebas unitarias se aplicarán a componentes individuales, evaluando su comportamiento de manera aislada.

Priorización

Se priorizarán módulos críticos del backend y frontend según su impacto funcional.

Automatización

El proceso será orientado a la automatización progresiva para facilitar el mantenimiento del software.

Responsables y cronograma

Rol Responsabilidad
Responsable de Calidad Coordinar la planificación de pruebas unitarias.
Desarrollador Backend Analizar módulos del servidor.
Desarrollador Frontend Analizar componentes de interfaz.
Responsable de Documentación Mantener actualizada la Wiki del proyecto.
Sprint Actividad principal
Sprint 1 Investigación, análisis del repositorio y elaboración del plan.
Sprint 2 Identificación detallada de componentes y preparación del entorno.
Sprint 3 Implementación inicial de pruebas unitarias.

Riesgos y consideraciones

Complejidad de arquitectura

Puede dificultar el aislamiento de componentes adecuados para pruebas unitarias.

Dependencias externas

Algunos módulos pueden requerir mecanismos de simulación o dobles de prueba.

Curva de aprendizaje

Las herramientas de testing pueden requerir tiempo adicional de adaptación.

Limitaciones de tiempo

El cronograma académico puede restringir el análisis técnico detallado.
